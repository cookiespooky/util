# Утилитсервис — текстовый сайт

Статический корпоративный сайт на Markdown и [Notepub](https://github.com/cookiespooky/notepub). Публичная версия собирается GitHub Actions и размещается на GitHub Pages.

## Публичный адрес

https://cookiespooky.github.io/util/

## Что внутри

- `content/` — тексты страниц с frontmatter и `[[wikilinks]]`;
- `theme/` — HTML-шаблоны, CSS и JavaScript;
- `rules.yaml` — типы страниц, маршруты, граф ссылок, поиск и sitemap;
- `config.yaml` — production-конфигурация GitHub Pages;
- `config.dev.yaml` — локальная конфигурация;
- `CONTENT-GLOSSARY.md` — единый словарь терминов и правила публичных формулировок;
- `CONTENT-SPEC.md` — структура меню, CTA, формы, заглушки и доверительные блоки;
- `scripts/build.sh` — воспроизводимая проверка и статическая сборка;
- `scripts/deploy.sh` — сборка, коммит и push текущей ветки;
- `.github/workflows/pages.yml` — автоматический деплой GitHub Pages.

## Локальная сборка

Требуется Go 1.22+ либо готовый бинарник `notepub` в `PATH`.

```bash
./scripts/build.sh
```

Скрипт:

1. устанавливает зафиксированную версию Notepub, если бинарника нет;
2. проверяет frontmatter, маршруты, wikilinks и Markdown;
3. строит индекс;
4. собирает `dist/`;
5. проверяет наличие обязательных файлов и локальные ссылки.

## Локальный просмотр

```bash
notepub serve --config ./config.dev.yaml --rules ./rules.yaml
```

Открыть http://127.0.0.1:8080/.

## Деплой

```bash
./scripts/deploy.sh "Update site content"
```

После push workflow автоматически пересобирает и публикует сайт.

## Статус контента

Подготовлена полная текстовая версия:

- главная страница;
- каталог и семь страниц услуг;
- страницы компании, контактов, FAQ, лицензий, клиентов и партнёров;
- страницы Сургута, Тюмени, Новосибирска и Нягани;
- единая терминология;
- структура меню и CTA;
- короткая общая форма и динамические поля по услугам;
- правила использования заглушек;
- структура доверительных блоков;
- системные сообщения для форм, поиска и страницы 404.

Все основные услуги считаются доступными во всех четырёх городах. Возможность выполнения конкретной операции определяется после проверки отхода, адреса, действующей лицензии и технических условий.

Количественные показатели, контакты, документы, логотипы клиентов и фотографии допускается подтверждать и заменять в процессе дизайна и вёрстки. Перед публикацией на основном домене требуется финальная проверка заказчиком.

Текстовый этап зафиксирован в `CONTENT-GLOSSARY.md` и `CONTENT-SPEC.md`. Следующий этап — дизайн компонентов и вёрстка.
